# 프로젝트 본질 정의

## 프로젝트명

시뮬레이션 기반 자동 밸런스 최적화 및 사용자 행동 분석 시스템

## 프로젝트 본질

전투 시뮬레이션과 사용자 로그 데이터를 기반으로 게임 밸런스를 자동으로 조정하고,
플레이 패턴을 분석하는 데이터 기반 의사결정 시스템이다.

## 프로젝트 목표

1. 전투 시뮬레이션을 통해 몬스터의 스탯을 자동으로 조정하여 목표 승률에 수렴하는 밸런스 시스템을 구축한다.
2. 플레이어의 전투 로그를 수집하고 분석하여 행동 패턴을 정량적으로 파악한다.
3. 분석된 데이터를 기반으로 자동전투 로직 및 플레이 피드백 시스템을 구현한다.

## 문제 정의

기존 게임 밸런싱은 개발자가 직접 플레이 테스트를 반복하며 수작업으로 수치를 조정해야 한다.
이 방식은 시간이 오래 걸리고, 경우의 수가 많아 최적값을 찾기 어렵고, 플레이어 스타일을 반영하기 어렵다.

## 해결 방법

1. Monte Carlo 기반 전투 시뮬레이션을 반복 실행하여 승률을 계산한다.
2. Binary Search를 이용해 목표 승률에 도달하는 몬스터 스탯을 자동으로 역산한다.
3. 플레이어 상태를 수치화한 Player Power Index를 통해 동적 난이도를 조정한다.
4. 전투 로그를 JSON 형태로 저장하여 행동 데이터를 수집한다.
5. 로그 데이터를 기반으로 플레이 패턴을 분석하고 규칙 기반 자동전투 및 피드백 시스템을 구성한다.

## 시스템 구조

```text
플레이어 행동
↓
전투 엔진 (Battle Engine)
↓
전투 로그 저장 (JSON)
↓
시뮬레이션 (Monte Carlo)
↓
밸런스 튜닝 (Binary Search)
↓
패턴 분석 (통계 기반)
↓
자동전투 / 피드백 생성
```

LLM은 로그를 자연어로 해석하는 보조 도구로만 사용한다.

## 핵심 구현 요소

- ATB 기반 전투 시스템
- 단일 전투 엔진
- Monte Carlo 시뮬레이션
- Binary Search 기반 밸런스 역산
- Player Power Index 기반 난이도 조정
- 멀티스레드 비동기 시뮬레이션
- 전투 로그 저장 및 분석 구조

## 핵심 메시지

이 프로젝트는 단순한 게임 구현이 아니라, 데이터와 시뮬레이션을 기반으로 의사결정을 수행하는
자동 밸런싱 및 사용자 행동 분석 엔진이다.

## 한 줄 요약

AI가 게임을 만드는 것이 아니라, 데이터를 기반으로 시스템이 스스로 판단하도록 설계한 프로젝트다.
